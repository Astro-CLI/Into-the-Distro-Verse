Apache + Tor Hidden Service with Docker

A complete guide to running an Apache web server as a Tor hidden service (.onion domain) using Docker.

ℹ What You'll Get: A hidden service accessible only via Tor, with easy cleanup (just delete the Docker containers).

Prerequisites

  • Docker installed
  • Docker Compose installed
  • Tor Browser (for testing)

Setup Instructions

Step 1: Create Working Directory

mkdir -p ~/apache-tor && cd ~/apache-tor

Step 2: Create Dockerfile

cat > Dockerfile << 'EOF' FROM httpd:latest COPY index.html /usr/local/apache2/htdocs/ EOF

Step 3: Create index.html

cat > index.html << 'EOF' <!DOCTYPE html> <html> <head><title>Onion Server</title></head> <body> <h1>Hello from .onion!</h1> <p>This is hosted via Tor hidden service</p> </body> </html> EOF

Step 4: Create Torrc Configuration

cat > torrc << 'EOF' SocksPort 0 ControlPort 9051 CookieAuthentication 1 HiddenServiceDir /var/lib/tor/hidden_service/ HiddenServicePort 80 apache:80 Log notice file /var/log/tor/notices.log EOF

Step 5: Create docker-compose.yml

cat > docker-compose.yml << 'EOF' version: '3' services: apache: build: . container_name: apache-server ports: - "8080:80" networks: - tornet tor: image: torproject/tor:latest container_name: tor-hidden-service volumes: - ./torrc:/etc/tor/torrc - tor-data:/var/lib/tor networks: - tornet depends_on: - apache networks: tornet: volumes: tor-data: EOF

Start the Services

docker-compose up -d

Get Your .onion Address

docker exec tor-hidden-service cat /var/lib/tor/hidden_service/hostname
Output Example: abcd1234efgh5678ijkl9999nnnn5555.onion

Access Your Server

Option 1: Via Tor Browser (Recommended)

  1. Download and install Tor Browser
  2. Open Tor Browser
  3. Visit: http://YOUR_ONION_ADDRESS.onion

Option 2: Via Torsocks (CLI)

torsocks curl http://YOUR_ONION_ADDRESS.onion

Option 3: Local HTTP Access

curl http://localhost:8080

Customization

Mount Custom Apache Config

Create httpd.conf and add to docker-compose.yml:

apache: volumes: - ./httpd.conf:/usr/local/apache2/conf/httpd.conf

Add HTTPS to .onion

Update torrc:

HiddenServicePort 443 apache:443

Custom Port Mapping

apache: ports: - "3000:80" # Access via http://localhost:3000

Stop & Cleanup

Stop Services (Keep Data)

docker-compose down

Stop & Remove Everything

docker-compose down -v # -v removes volumes

Complete Cleanup

cd .. rm -rf ~/apache-tor

Useful Commands

Command Purpose
docker-compose logs tor View Tor logs
docker-compose logs apache View Apache logs
docker exec tor-hidden-service Run commands inside Tor container
docker ps List running containers

Security Notes

Important:
  • Your .onion address is YOUR private key — keep it secret
  • Don't expose the tor-data volume outside the project
  • Tor hidden services are anonymous, but your content is visible to everyone with the address
  • Always run containers with minimal privileges

Troubleshooting

Port Already in Use

sudo lsof -i :8080 kill -9 <PID>

Tor Won't Start

docker-compose logs tor

Apache Returns 404

docker exec apache-server ls /usr/local/apache2/htdocs/

Can't Access .onion Address

  • Wait 30-60 seconds after startup (Tor needs time to bootstrap)
  • Ensure Tor Browser is connected
  • Check hostname file exists: docker exec tor-hidden-service cat /var/lib/tor/hidden_service/hostname
